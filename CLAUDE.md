# Schnittstellenpass Project Architecture

## Project Overview

**Project Name:** Schnittstellenpass  
**Framework:** Angular 17.3.0  
**Application Type:** Standalone Angular Application (no NgModule-based architecture)  
**Build Tool:** Angular CLI 17.3.0  
**Styling:** SCSS + Tailwind CSS  
**Testing:** Karma + Jasmine

## Application Architecture

### Bootstrap Process
1. **main.ts** - Entry point that calls `bootstrapApplication(AppComponent, appConfig)`
2. **AppComponent** - Standalone root component with RouterOutlet for application routing
3. **appConfig** - Provides routing configuration via `provideRouter(routes)`

### Standalone Architecture Pattern
This project uses Angular 17's standalone components approach, eliminating the need for NgModules:
- Components use `standalone: true` flag
- Dependencies are provided via `provideRouter()` in the application config
- No traditional module files (no app.module.ts)
- Direct dependency injection using providers in ApplicationConfig

### Routing
- **app.routes.ts**: Currently empty - routes can be added here as features develop
- **Router Outlet**: Present in app.component.html for rendering routed components
